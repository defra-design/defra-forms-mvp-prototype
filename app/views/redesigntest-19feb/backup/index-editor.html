Draft status
When a form is created it automatically adopt a status of ‘Draft’. In Draft status, the user can edit and make a form live.

  • View a coded overview page in draft status
  • View an overview page in draft where contact support has not been added

Live status
When a user has provided required information, the user will then see on their Overview page, a button labelled ‘Make draft live’.

  • View a coded overview page in live status

Draft and Live (creating a draft to edit)
  • to avoid the complexity of creating multiple versions of the same form, users can only edit a form when it’s in ‘Draft’ status
  • the only way to edit a form that’s ‘Live’ is to create a ‘Draft’ version of it
  • once you’ve made changes to a draft version of a live form, when you make the draft version live, it replaces the existing Live form
  • View a coded overview page in draft and live status

Replacing a live form with a draft
Users are warned that making a draft live while there is already a live form means that the existing live form will be replaced with the draft.
  • View a coded warning page for replacing a live form
